# 教育学习助手系统需求规格说明书 SRS

## 1. 项目背景

家庭教育中，家长通常能看到孩子作业和考试结果，但很难系统判断孩子到底哪些知识点掌握不好。教材、课堂、作业、错题之间没有形成数据闭环。

当前主要问题包括：

1. 教材内容没有结构化沉淀。
2. 课堂视频看完后难以复盘。
3. 错题分析依赖人工经验。
4. 无法持续跟踪知识点掌握趋势。
5. 学习方案缺少数据支撑。

系统目标是用 AI 技术把教材、课堂和错题连接起来，形成孩子的学习画像和可执行学习方案。

## 2. 项目目标

建设一个本地优先的教育学习辅助系统，支持：

1. 教材内容识别与知识点沉淀。
2. 作业与错题识别、归因和分析。
3. 知识掌握度评估。
4. 学习报告和学习方案生成。
5. 后续支持课堂视频学习、智能问答和 OpenClaw 接入。

## 3. 用户角色

### 3.1 家长

家长是主要使用者，负责上传教材、上传作业、查看报告、调整辅导策略。

### 3.2 学生

学生用于查看知识点解释、错题讲解和学习计划。

### 3.3 系统管理员

用于配置学科、年级、教材版本、模型参数和存储路径。

### 3.4 AI服务

自动完成OCR、知识点提取、错题分析、报告生成。

## 4. 系统范围

### 4.1 V1包含范围

1. 教材图片上传。
2. OCR识别。
3. 教材知识点提取。
4. 作业/错题图片上传。
5. 题目识别。
6. 错因分析。
7. 知识点匹配。
8. 掌握度评估。
9. Markdown报告生成。
10. 本地落盘。

### 4.2 V1不包含范围

1. 完整课堂视频学习。
2. 完整知识图谱可视化。
3. 双角色智能问答。
4. PDF错题本导出。
5. 复杂多用户权限。
6. Electron桌面端。

## 5. 功能需求

## 5.1 教材管理

### 5.1.1 新增教材

系统应支持录入教材信息：

```text
教材名称
年级
学科
版本
出版社
册次
备注
```

### 5.1.2 上传教材页面

系统应支持上传教材图片或PDF。

### 5.1.3 教材识别

系统应对教材页面进行OCR识别，保存原始图片、OCR文本和结构化结果。

### 5.1.4 知识点提取

系统应从教材内容中提取：

```text
章节
小节
知识点
定义
公式
例题
习题
易错点
学习目标
```

### 5.1.5 教材知识落盘

系统应将识别结果保存为：

```text
JSON结构化文件
Markdown知识文档
SQLite记录
向量索引
```

## 5.2 作业与错题分析

### 5.2.1 上传作业

系统应支持上传单题图片、整页作业图片、试卷图片。

### 5.2.2 题目识别

系统应识别题干、选项、图形说明、公式和题号。

### 5.2.3 学生答案识别

系统应识别学生手写或印刷答案。

### 5.2.4 批改结果识别

系统应识别老师批改痕迹，如勾、叉、分数、订正标记。

### 5.2.5 错误分析

系统应对错题进行错因归类：

```text
概念不清
公式记错
审题错误
计算错误
步骤缺失
方法不会
知识点混淆
迁移能力不足
粗心错误
```

### 5.2.6 知识点匹配

系统应将错题关联到教材知识点。

### 5.2.7 错题记录

系统应记录：

```text
题目原图
OCR文本
学生答案
正确答案
错因
知识点
订正建议
分析时间
置信度
```

## 5.3 掌握度评估

系统应根据错题数量、知识点出现频率、重复错误、订正情况计算掌握度。

掌握度分级：

```text
90-100：掌握较好
75-89：基本掌握
60-74：掌握不稳定
40-59：明显薄弱
0-39：未掌握
```

系统应支持按以下维度查看掌握度：

```text
学生
学科
年级
教材
章节
知识点
时间范围
```

## 5.4 学习报告

系统应生成 Markdown 学习报告，内容包括：

1. 本次分析概况。
2. 知识点掌握情况。
3. 薄弱知识点。
4. 错因统计。
5. 典型错题。
6. 下一步学习建议。
7. 建议复习路径。

## 5.5 学习方案

系统应根据薄弱知识点生成学习方案：

```text
阶段1：回看教材概念
阶段2：完成基础题训练
阶段3：完成变式题训练
阶段4：复盘错题
阶段5：再次检测
```

## 5.6 本地知识库

系统应保存以下数据：

```text
原始图片
OCR文本
结构化知识点
错题记录
学生画像
学习报告
运行日志
```

## 6. 非功能需求

### 6.1 本地优先

默认所有数据保存在本地，不强制上传云端。

### 6.2 可配置

模型、存储路径、学科体系、掌握度规则应可配置。

### 6.3 可追溯

所有AI分析结果应保留原始输入、分析依据和置信度。

### 6.4 可人工修正

OCR结果、知识点匹配、错因分析应支持人工修改。

### 6.5 可扩展

后续应支持课堂视频、智能问答、知识图谱、OpenClaw接入。

## 7. 验收标准

V1验收标准：

1. 能上传教材图片并生成OCR文本。
2. 能从教材图片中提取知识点。
3. 能上传作业/错题图片。
4. 能识别题目内容。
5. 能给出错因分析。
6. 能关联至少一个知识点。
7. 能生成掌握度报告。
8. 能将结果保存到本地目录。
9. 能通过API查询分析结果。
10. 能输出Markdown报告。

## 8. 关键约束

1. V1先不追求完美OCR，允许低置信度标记。
2. V1先支持小学数学和英语试点。
3. V1不做复杂前端，以API和简单页面验证为主。
4. 所有AI分析结果不得作为唯一教育结论，应作为辅助建议。
