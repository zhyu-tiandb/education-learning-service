# 教育学习辅助系统设计文档

**项目名称：** 基于大模型的小初高学习辅助系统  
**设计日期：** 2024-04-24  
**设计阶段：** 系统架构设计  
**试点范围：** 小学二年级数学 + 新东方英语

---

## 一、项目概述

### 1.1 项目愿景

构建一个本地优先、云端增强的智能教育辅助系统，通过识别教材内容、构建知识图谱、智能问答和作业分析，帮助家长了解孩子的学习情况，提供个性化学习建议。

### 1.2 核心功能

1. **教材识别与学习**：通过高拍仪拍摄教材或导入电子书，自动识别并提取知识点
2. **知识图谱构建**：将教材内容结构化，构建知识点关联关系
3. **智能问答**：支持家长和学生双角色问答，基于知识库提供个性化解答
4. **作业分析**：拍摄或输入孩子作业，自动批改并分析薄弱知识点
5. **学习报告**：生成知识掌握情况报告，提供学习建议

### 1.3 目标用户

- **主要用户**：家长（管理教材、查看学习报告、获取辅导建议）
- **次要用户**：学生（智能问答、查询不懂的知识点）

## 三、数据模型设计

### 3.1 核心实体

```
教材 (Textbook)
├── 章目 (Chapter)
│   └── 知识点 (KnowledgePoint)
│       ├── 知识点定义
│       ├── 例题 (Example)
│       ├── 习题 (Exercise)
│       └── 关联关系 (Relation)
├── 页面 (Page)
│   ├── 原始图像
│   ├── OCR文本
│   └── 解析后的结构化内容
└── 元数据 (年级、学科、版本、出版社)

作业 (Homework)
├── 作业题目 (HomeworkQuestion)
│   ├── 题目内容
│   ├── 学生答案
│   ├── 批改结果
│   └── 错误分析
└── 掌握报告 (MasteryReport)

学习记录 (LearningRecord)
├── 学生信息
├── 学习进度
└── 掌握程度
```

### 3.2 知识图谱结构

**知识点属性：**
- 名称、描述、难度等级
- 所属章节、学科
- 类型：概念/技能/公式

**关联关系类型：**
- `PREREQUISITE` - 前置知识
- `RELATED_TO` - 相关知识点
- `PART_OF` - 属于某章节
- `EXAMPLED_BY` - 有例题演示

## 四、图像处理与OCR流程

### 4.1 处理流程

```
高拍仪拍摄/电子书导入
    │
    ▼
图像预处理
├── 去噪、校正倾斜
├── 自动裁剪（检测教材边界）
└── 图像增强
    │
    ▼
OCR识别
├── 文本检测（PaddleOCR）
├── 文本识别（中英文）
└── 版面分析（标题、正文、公式分离）
    │
    ▼
结构化提取
├── 章节层级识别
├── 知识点边界检测
└── 例题/习题分离
    │
    ▼
LLM语义增强
├── 知识点提取与命名
├── 关联关系识别
└── 难度等级标注
```

### 4.2 技术选型

| 组件 | 技术方案 | 说明 |
|------|---------|------|
| 文本检测/识别 | PaddleOCR | 开源、中英文支持好 |
| 版面分析 | PaddleOCR Layout | 教材排版分析 |
| 公式识别 | LaTeX-OCR (Pix2Tex) | 数学公式转换 |
| 语义理解 | 云端LLM API | 知识提取与关系识别 |

## 六、作业分析与学习报告

### 6.1 作业输入方式

- 拍摄输入：单题/整页/批量
- 手动输入：文本输入、公式编辑器
- 文件导入：图片、PDF

### 6.2 分析流程

```
作业输入
    │
    ▼
题目识别与提取
├── OCR文字识别
├── 手写内容提取
├── 题目类型分类
└── 知识点匹配
    │
    ▼
答案批改
├── 客观题自动判分
├── 主观题LLM辅助批改
├── 错误类型分析
└── 错误步骤定位
    │
    ▼
知识关联分析
├── 识别涉及的知识点
├── 判断知识点掌握程度
└── 发现薄弱环节
    │
    ▼
生成学习报告
```

### 6.3 报告内容

1. **总体评估**：正确率、用时、题目数量
2. **知识点掌握雷达图**：可视化展示各知识点掌握度
3. **薄弱知识点分析**：错误统计、类型分析、改进建议
4. **错题本**：可导出PDF打印重做
5. **学习建议**：基于分析的个性化建议

## 十、技术风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| OCR准确率不足 | 知识提取质量差 | 多模型融合、人工校正 |
| LLM成本过高 | 运营成本高 | 本地缓存、智能去重 |
| 手写识别困难 | 作业分析受限 | 引导用户规范书写 |
| 数据隐私顾虑 | 用户接受度低 | 本地优先存储、可配置云服务 |
