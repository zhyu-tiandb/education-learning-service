# 教育学习助手系统数据模型设计

## 1. 设计原则

1. 原始数据与AI分析结果分离。
2. 所有分析结果保留证据来源。
3. 数据库记录索引，详细内容可保存为JSON/Markdown。
4. 支持后续知识图谱和向量检索扩展。

## 14. 完整SQL建表语句

### 14.1 学生表

```sql
CREATE TABLE students (
    student_id VARCHAR(50) PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    grade VARCHAR(20) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_students_grade ON students(grade);
```

### 14.2 教材表

```sql
CREATE TABLE textbooks (
    textbook_id VARCHAR(50) PRIMARY KEY,
    name VARCHAR(200) NOT NULL,
    subject VARCHAR(50) NOT NULL,
    grade VARCHAR(20) NOT NULL,
    publisher VARCHAR(100),
    volume VARCHAR(20),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(textbook_id, grade, subject, volume)
);

CREATE INDEX idx_textbooks_subject_grade ON textbooks(subject, grade);
```

### 14.3 教材页面表

```sql
CREATE TABLE textbook_pages (
    page_id VARCHAR(50) PRIMARY KEY,
    textbook_id VARCHAR(50) NOT NULL,
    page_no INTEGER NOT NULL,
    chapter VARCHAR(100),
    section VARCHAR(100),
    image_path VARCHAR(500) NOT NULL,
    ocr_text_path VARCHAR(500),
    structured_json_path VARCHAR(500),
    ocr_confidence DECIMAL(3,2),
    status VARCHAR(20) NOT NULL DEFAULT 'PENDING',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (textbook_id) REFERENCES textbooks(textbook_id),
    UNIQUE(textbook_id, page_no)
);

CREATE INDEX idx_textbook_pages_textbook ON textbook_pages(textbook_id);
CREATE INDEX idx_textbook_pages_status ON textbook_pages(status);
CREATE INDEX idx_textbook_pages_chapter ON textbook_pages(chapter);
```

### 14.4 知识点表

```sql
CREATE TABLE knowledge_points (
    knowledge_point_id VARCHAR(50) PRIMARY KEY,
    subject VARCHAR(50) NOT NULL,
    grade VARCHAR(20) NOT NULL,
    textbook_id VARCHAR(50),
    chapter VARCHAR(100),
    section VARCHAR(100),
    name VARCHAR(200) NOT NULL,
    type VARCHAR(20) NOT NULL,
    definition TEXT,
    difficulty VARCHAR(20) NOT NULL DEFAULT '基础',
    tags JSON,
    source_page_id VARCHAR(50),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (textbook_id) REFERENCES textbooks(textbook_id),
    FOREIGN KEY (source_page_id) REFERENCES textbook_pages(page_id)
);

CREATE INDEX idx_knowledge_points_subject_grade ON knowledge_points(subject, grade);
CREATE INDEX idx_knowledge_points_textbook ON knowledge_points(textbook_id);
CREATE INDEX idx_knowledge_points_chapter ON knowledge_points(chapter);
CREATE INDEX idx_knowledge_points_type ON knowledge_points(type);
```

### 14.5 知识点关系表

```sql
CREATE TABLE knowledge_relations (
    relation_id VARCHAR(50) PRIMARY KEY,
    source_kp_id VARCHAR(50) NOT NULL,
    target_kp_id VARCHAR(50) NOT NULL,
    relation_type VARCHAR(20) NOT NULL,
    confidence DECIMAL(3,2),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (source_kp_id) REFERENCES knowledge_points(knowledge_point_id),
    FOREIGN KEY (target_kp_id) REFERENCES knowledge_points(knowledge_point_id),
    UNIQUE(source_kp_id, target_kp_id, relation_type)
);

CREATE INDEX idx_knowledge_relations_source ON knowledge_relations(source_kp_id);
CREATE INDEX idx_knowledge_relations_target ON knowledge_relations(target_kp_id);
CREATE INDEX idx_knowledge_relations_type ON knowledge_relations(relation_type);
```

### 14.6 作业记录表


### 14.7 错题表

```sql
CREATE TABLE wrong_questions (
```

### 14.8 掌握度记录表

```sql
CREATE TABLE mastery_records (
```

## 15. 索引设计策略

### 15.1 索引设计原则

1. **高频查询字段优先索引**：student_id、knowledge_point_id、created_at
2. **组合索引优化**：subject + grade、student_id + subject
3. **外键字段建立索引**：提升JOIN性能
4. **状态字段建立索引**：status、level等枚举字段

### 15.2 查询场景优化

| 查询场景 | 使用的索引 |
|---------|----------|
| 学生查询某学科错题 | idx_wrong_questions_student + idx_wrong_questions_error_type |
| 知识点掌握度查询 | idx_mastery_student + idx_mastery_score |
| 时间范围报告查询 | idx_reports_period + idx_reports_created |
| 任务状态监控 | idx_jobs_status + idx_jobs_created |
| 教材章节内容 | idx_textbook_pages_textbook + idx_textbook_pages_chapter |

## 17. 向量检索方案

### 17.1 向量化内容

**需要向量化的数据：**

| 数据类型 | 向量化字段 | 用途 |
|---------|----------|------|
| 知识点 | name + definition | 知识点语义检索 |
| 错题 | question_text + error_reason | 相似错题推荐 |
| 例题 | question_text | 题目相似度检索 |

### 17.4 检索策略

```python
# 语义检索示例
def search_similar_knowledge(query: str, top_k: int = 5):
    query_embedding = model.encode(query)
    results = knowledge_collection.query(
        query_embeddings=[query_embedding],
        n_results=top_k
    )
    return results

# 混合检索：向量相似度 + 结构化过滤
def search_knowledge_with_filter(
    query: str,
    subject: str = None,
    grade: str = None,
    top_k: int = 5
):
    query_embedding = model.encode(query)
    results = knowledge_collection.query(
        query_embeddings=[query_embedding],
        n_results=top_k,
        where={
            "subject": subject,
            "grade": grade
        } if subject and grade else None
    )
    return results
```

## 18. ER关系说明

### 18.1 核心实体关系

```text
学生 (Student)
  ├─ 1:N → 作业记录 (HomeworkRecord)
  ├─ 1:N → 错题 (WrongQuestion)
  ├─ 1:N → 掌握度记录 (MasteryRecord)
  └─ 1:N → 学习报告 (LearningReport)

教材 (Textbook)
  ├─ 1:N → 教材页面 (TextbookPage)
  └─ 1:N → 知识点 (KnowledgePoint)

教材页面 (TextbookPage)
  └─ 1:N → 知识点 (KnowledgePoint)

知识点 (KnowledgePoint)
  ├─ 1:N → 知识点关系 (KnowledgeRelation) [作为起点]
  ├─ 1:N → 知识点关系 (KnowledgeRelation) [作为终点]
  ├─ 1:N → 掌握度记录 (MasteryRecord)
  └─ N:M → 错题 (WrongQuestion) [通过JSON数组]

作业记录 (HomeworkRecord)
  └─ 1:N → 错题 (WrongQuestion)
```

### 18.2 关系类型说明

| 关系 | 类型 | 说明 |
|------|------|------|
| Student → HomeworkRecord | 1:N | 一个学生有多条作业记录 |
| HomeworkRecord → WrongQuestion | 1:N | 一份作业可能有多个错题 |
| Student → MasteryRecord | 1:N | 一个学生对多个知识点有掌握度记录 |
| KnowledgePoint → MasteryRecord | 1:N | 一个知识点被多个学生记录掌握度 |
| Textbook → TextbookPage | 1:N | 一本教材有多页 |
| TextbookPage → KnowledgePoint | 1:N | 一页可能有多个知识点 |
| KnowledgePoint → KnowledgeRelation | 自关联N:N | 知识点之间的多种关系 |

### 18.3 数据流示意图

```text
[教材图片] → TextbookPage → OCR识别 → [知识抽取] → KnowledgePoint
                                              ↓
[作业图片] → HomeworkRecord → OCR识别 → [错题分析] → WrongQuestion
                                              ↓
                                         [掌握度计算] → MasteryRecord
                                              ↓
                                         [报告生成] → LearningReport
```

### 18.4 关键约束

1. **唯一约束**
   - 每个学生对每个知识点只能有一条掌握度记录
   - 同一教材的同一页码只能有一条记录

2. **级联规则**
   - 删除教材时级联删除页面和知识点（V1简化处理）
   - 删除作业时级联删除错题记录
   - 删除学生时需要处理关联的掌握度和报告

3. **数据一致性**
   - mastery_records.score 更新时，level 应同步更新
   - wrong_questions 修改时，相关的 mastery_records 应更新
