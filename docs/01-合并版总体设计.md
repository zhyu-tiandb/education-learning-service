# 教育学习助手系统合并版总体设计

## 1. 系统定位

本系统定位为一个“本地优先、云端增强、可独立运行、后续可接入 OpenClaw”的教育学习辅助系统。

系统不是简单的拍照搜题，也不是单纯的 OCR 工具，而是围绕孩子长期学习过程形成闭环：

```text
教材知识库
+
课堂经验库
+
错题/作业记录库
+
知识掌握画像
+
学习报告与学习方案
```

系统第一阶段应优先解决“看得见、存得住、分析得出、报告可用”的问题。

## 2. 建设目标

### 2.1 教材自动学习

通过高拍仪、教材照片或 PDF，自动识别教材内容，提取章节、知识点、定义、公式、例题、习题、易错点，并沉淀到本地知识库。

### 2.2 课堂内容自动学习

通过课堂视频、录播视频或网课视频，自动识别老师讲解内容、板书/PPT内容、重点提示、例题讲解和易错提醒，并与教材知识点关联。

课堂视频处理成本高，建议放在 V2 阶段，不进入 V1。

### 2.3 错题与作业分析

通过拍摄作业、试卷、错题本，识别题目、学生答案和批改结果，分析错误原因，判断知识掌握情况，并生成学习报告和学习方案。

                       │

## 4. 技术路线

### 4.1 后端

```text
Python FastAPI
SQLite
SQLAlchemy / Pydantic
ChromaDB 或 FAISS
Markdown + JSON 本地文件
```

### 4.2 前端

```text
React + TypeScript
shadcn/ui
Tailwind CSS
Recharts
Zustand
```

V1可以不做完整前端，先通过 API + 简单页面验证闭环。

### 4.3 AI能力

```text
OCR：PaddleOCR
版面分析：PaddleOCR Layout
公式识别：Pix2Tex / LaTeX-OCR
语音识别：Whisper
大模型：本地模型或云端LLM API
向量检索：ChromaDB / FAISS
```

### 4.4 部署方式

```text
第一阶段：本地服务 + API
第二阶段：Web前端
第三阶段：Electron桌面端
第四阶段：OpenClaw Skill接入
```

## 5. 功能模块

### 5.1 教材学习模块

输入：

```text
高拍仪图片
手机拍摄图片
教材PDF
电子书文件
```

功能：

```text
图片预处理
OCR识别
版面结构分析
章节层级识别
公式识别
例题/习题分离
知识点抽取
易错点提取
知识点关系识别
```

输出：

```text
教材名称
年级
学科
章节
知识点
定义
公式
例题
习题
解题方法
易错点
前置知识
关联知识点
```

### 5.2 课堂视频学习模块

输入：

```text
课堂录播视频
网课视频
手机拍摄视频
```

功能：

```text
视频抽帧
音频提取
语音识别
板书/PPT识别
讲解内容转写
重点内容提取
例题讲解提取
与教材知识点关联
```

V1不实现，V2实现。

### 5.3 作业与错题分析模块

输入：

```text
作业照片
试卷照片
错题本照片
手写答案
批改痕迹
手工录入题目
```

功能：

```text
题目识别
答案识别
批改结果识别
题型判断
知识点匹配
正误判断
错误步骤定位
错因分析
错题归档
```

错因分类：

```text
概念不清
公式记错
审题错误
计算错误
步骤缺失
方法不会
知识点混淆
迁移能力不足
粗心错误
```

### 5.4 知识图谱与知识库模块

知识点属性：

```text
知识点名称
知识点描述
所属学科
所属年级
所属教材
所属章节
难度等级
知识点类型：概念/技能/公式/方法/题型
```

知识关系：

```text
PREREQUISITE：前置知识
RELATED_TO：相关知识
PART_OF：属于章节
EXAMPLED_BY：例题演示
CONFUSED_WITH：易混淆知识
TESTED_BY：常见考查题型
```

V1仅做知识点标签和简单关系，V2再做完整知识图谱。

### 5.5 学生画像与掌握度评估模块

数据来源：

```text
错题记录
作业正确率
重复错误次数
知识点出现频率
订正情况
复习完成情况
```

掌握度分级：

```text
90-100：掌握较好
75-89：基本掌握
60-74：掌握不稳定
40-59：明显薄弱
0-39：未掌握
```

### 5.6 智能问答模块

角色模式：

```text
家长模式：
解释孩子薄弱点、辅导建议、学习计划、错题复盘方法。

学生模式：
用更简单的语言解释知识点，引导理解，不直接只给答案。
```

V1不作为主线，V2实现。

### 5.7 学习报告模块

报告内容：

```text
总体学习情况
本阶段完成内容
知识点掌握分布
薄弱知识点分析
错因统计
典型错题
重复错误提醒
学习建议
下一阶段学习计划
```

输出形式：

```text
Markdown
HTML
PDF
错题本PDF
周报
月报
```

V1优先输出 Markdown。

## 6. V1实施范围

V1必须实现：

```text
教材图片上传
教材OCR识别
教材知识点提取
错题/作业图片上传
错题识别
知识点匹配
错因分析
掌握度评估
Markdown学习报告
本地文件落盘
SQLite记录
```

V1暂不实现：

```text
完整React前端
课堂视频理解
复杂知识图谱可视化
双角色智能问答
PDF错题本导出
多学生复杂权限
Electron桌面端
```

## 7. 演进路线

### V1：本地服务闭环

目标：先跑通教材与错题闭环。

```text
教材识别 → 知识点沉淀 → 错题分析 → 掌握度评估 → 学习报告
```

### V1.5：产品化界面

目标：让家长可以通过页面使用。

```text
React前端
教材管理
作业上传
知识库浏览
报告查看
错题列表
```

### V2：智能增强

目标：形成真正的学习辅助系统。

```text
知识图谱
双角色问答
课堂视频学习
学习画像趋势
错题本PDF
学习计划跟踪
```

### V3：长期学习系统

目标：形成长期个性化学习助手。

```text
多学生支持
多学科支持
阶段性能力评估
复习提醒
推荐练习
OpenClaw Skill接入
```

## 8. 总体结论

合并后的方案采用：

```text
产品化总体架构 + V1最小闭环落地
```

实施上不能一开始就做完整大系统，应先完成：

```text
教材OCR + 知识点提取 + 错题分析 + 掌握度报告
```

等闭环跑通后，再扩展：

```text
前端界面 → 知识图谱 → 智能问答 → 课堂视频 → 长期画像
```
