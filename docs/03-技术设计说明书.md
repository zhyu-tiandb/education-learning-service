# 教育学习助手系统技术设计说明书

## 1. 技术总体原则

系统采用本地优先架构，尽量将原始数据、结构化知识和学生画像保存在本地。

设计原则：

1. 先闭环，后复杂。
2. 先本地，后云端。
3. 先文件可见，后数据库增强。
4. 先可人工修正，后自动化增强。
5. 所有AI结论必须可追溯。

## 2. 技术架构

```text
前端层：
React / 简单HTML页面 / 后续Electron

后端层：
FastAPI + Python服务模块

AI能力层：
PaddleOCR + LLM + 向量检索 + Whisper

存储层：
SQLite + Markdown + JSON + 原始图片 + ChromaDB/FAISS

扩展层：
OpenClaw Skill + 云端LLM + 桌面端
```

## 4. 数据处理流程

## 4.1 教材学习流程

```text
教材图片上传
    ↓
保存原图
    ↓
图片预处理
    ↓
OCR识别
    ↓
版面结构识别
    ↓
LLM提取知识点
    ↓
生成Markdown知识文档
    ↓
写入SQLite
    ↓
写入向量索引
```

## 4.2 错题分析流程

```text
错题图片上传
    ↓
保存原图
    ↓
OCR识别题目和答案
    ↓
题目结构化
    ↓
匹配教材知识点
    ↓
LLM分析错因
    ↓
更新掌握度
    ↓
保存错题记录
    ↓
生成学习建议
```

## 4.3 报告生成流程

```text
读取错题记录
    ↓
按知识点聚合
    ↓
计算掌握度
    ↓
识别薄弱知识点
    ↓
统计错因分布
    ↓
生成学习方案
    ↓
输出Markdown报告
```

## 5. AI能力设计

### 5.1 OCR

V1采用 PaddleOCR。

OCR结果结构：

```json
{
  "text": "识别文本",
  "confidence": 0.92,
  "bbox": [0, 0, 100, 30],
  "page_id": "page_xxx"
}
```

### 5.2 LLM知识点提取

输入：OCR文本 + 教材元数据。

输出：

```json
{
  "chapter": "小数乘法",
  "knowledge_points": [
    {
      "name": "小数乘整数",
      "type": "计算方法",
      "definition": "先按整数乘法计算，再确定小数点位置",
      "examples": [],
      "common_mistakes": []
    }
  ]
}
```

### 5.3 错因分析

输入：题目、学生答案、正确答案、相关知识点。

输出：

```json
{
  "is_wrong": true,
  "error_type": "概念不清",
  "error_reason": "未理解小数位数与结果小数点位置的关系",
  "related_knowledge_points": ["小数乘整数"],
  "suggestion": "先复习教材例题，再做10道基础题"
}
```

### 5.4 掌握度评分

初始规则建议：

```text
知识点初始分：80
每次相关错题：-5
重复同类错误：额外 -5
订正完成：+3
连续正确：+5
最低0，最高100
```

后续可根据实际效果调整。

## 8. 任务状态机

所有耗时任务统一采用状态机。

```text
PENDING：等待处理
RUNNING：处理中
SUCCESS：处理成功
FAILED：处理失败
NEED_REVIEW：需要人工确认
```

任务表字段：

```text
job_id
job_type
input_path
status
progress
error_message
created_at
updated_at
```

## 10. 工程边界

V1不要求：

1. OCR达到100%准确。
2. 所有题型自动批改。
3. 完整视频理解。
4. 完整知识图谱。
5. 精美前端。

V1必须保证：

1. 数据能进入系统。
2. 处理过程可追踪。
3. 结果能落盘。
4. 报告能生成。
5. 错误能定位。
