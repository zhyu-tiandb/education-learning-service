# 教育学习助手系统 - 企业级落地设计包

## 文档用途

本设计包用于将“教育学习助手”从想法推进到可研发、可评审、可验收的工程项目。

## 包含文档

1. `01-合并版总体设计.md`  
   系统定位、总体架构、模块边界、技术路线、版本演进。

2. `02-需求规格说明书-SRS.md`  
   业务目标、用户角色、功能需求、非功能需求、MVP范围。

3. `03-技术设计说明书.md`  
   后端、前端、AI能力、本地存储、知识库、数据流设计。

4. `04-API接口设计.md`  
   V1阶段后端接口设计，便于前后端并行开发。

5. `05-研发任务拆解与里程碑.md`  
   按模块拆成研发任务，明确输入、输出、依赖和验收点。

6. `06-评审检查表.md`  
   需求评审、设计评审、开发完成评审、测试验收检查表。

7. `07-数据模型设计.md`  
   教材、知识点、作业、错题、学生画像、报告的数据结构。

8. `08-OpenClaw接入设计.md`  
   后续如何通过 OpenClaw Skill 调用本地服务、沉淀经验和生成报告。

## 推荐实施顺序

第一步：只做 V1 本地服务闭环。

```text
教材图片上传 → OCR识别 → 知识点提取 → 错题分析 → 掌握度评估 → 学习报告 → 本地落盘
```

第二步：再做前端页面。

第三步：再做知识图谱、课堂视频、智能问答和 OpenClaw 接入。
